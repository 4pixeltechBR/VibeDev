## 🇺🇸 English

**Release date:** 2026-07-11
**Scope:** VibeDev core (v1.5.0)

When we audited v3.3.0 by simulating a total beginner (Carlos, 38, Instagram reel, zero programming experience), we found **8 new gaps** in the layman flow. This release closes 7 of them, plus generalizes the Brazilian Mode into a **Country Mode** covering 12 countries.

The 3 prior releases (v3.1.0 → v3.3.0) solved "understand" (glossary, cost, kill), "sustain" (time, creep, recognition), and "launch" (spark, launch, Brasil). v3.4.0 solves **"I arrived, I have no idea, I'm lost"** — the very first 5 minutes of the layman lifecycle that we hadn't formalized.

### What's in this release

#### 2. Onboarding Tour (3 Messages)

New file `references/onboarding-leigo.md`.

For total beginners arriving at the framework cold (post-Instagram-reel), the `/vd-status` 4-line technical format is meaningless. The tour runs **only on the first session of a new project, only in leigo mode**, and shows 3 short messages:

1. **Honest recognition** — "we don't pretend to know how the idea becomes an app. This is step-by-step construction with you deciding."
2. **Visual map of the 5 phases** — ASCII diagram showing the path.
3. **Binary choice to start** — "are you starting from zero or rescuing something messy?"

After the third message, the framework marks `onboarding_completo: true` in the state and **never** repeats the tour.

**Why 3 messages and not 10:** leigos don't read walls of text. Three messages respects attention budget.

#### 3. Painel Visual de Progresso (Visual Progress Panel)

New file `references/painel-progresso.md`.

Replaces `/vd-status` output when `modo_usuario: leigo`. Shows:

- Project name at top
- Trail (Verde / Vermelha)
- Progress bar (5 phases total) with current phase highlighted
- Checklist of phases with state (✅ done / ➔ current / ⏸ future)
- Current sub-task + number / total
- Last milestone achieved (from post-mortem)
- Estimated monthly cost (if Phase 3 already passed)
- Next concrete step

**Format:** ASCII pure (Unicode box drawing and emojis). Works in any terminal. Color is **optional** — handled with ANSI codes but skipped gracefully on plain terminals.

**Doesn't replace the state file** — the panel is a reader, not the source of truth.

#### 4. Decisão Confiante por Padrão (Confident Default Decision)

Modification in `SKILL.md` → "Regras universais".

Leigos tend to say "escolhe você" when faced with Type 1 technical options. The framework used to respect this as delegation, but **registered no justification** — leading to regretted decisions later.

New behavior:

1. IA presents Type 1 options in plain language
2. If user says "you choose" / "doesn't matter" / "I trust you":
   - IA picks the Red-Team recommended option
   - Justifies in 1 line
   - Records "decision Type X delegated to IA by user on [date]" in Decision Log
   - Continues
3. If user says "I want to understand better":
   - IA explains the trade-off in layman terms with 1 analogy if useful
   - Doesn't ask again unless requested

**Records delegation explicitly** — so future sessions know that the user didn't understand the original decision.

#### 5. Validação Anti-"Tá" (Anti-"It's Fine" Validation)

Modification in `/vd-check`.

Leigos say "tá" / "should be fine" / "I think so" to end the session quickly. Accepting those as approval means shipping untested features.

New protocol:

1. `/vd-check` asks a specific observable question:
   > "Did you click on '[button X]', filled the fields, and saw [specific visible result] appear? Yes or no — no guessing."
2. If user answers vaguely:
   - **Not accepted as approval**
   - Reformulation: "Did this happen NOW, on your screen, a moment ago?"
   - If still vague → **DEBUG mode** (ask for screenshot, log, or specific observation)
3. Only explicit confirmation moves the sub-task to done state

**Protects the framework from false positives** that would otherwise accumulate into "the system works but I can't actually verify it" situations.

#### 6. Recap Automático (Automatic Recap When Returning)

New file `references/recap-automatico.md`.

Leigos disappear for days or weeks. When they return, the framework used to load the state and ask "where do you want to continue?" — cold, mechanical, unhelpful.

New behavior in `/vd-status` when `data_atual - ultima_sessao_em ≥ 7 days`:

- Expand panel into **recap estendido** (welcome back + last state + decisions made + persona + options)
- Present 3 explicit options: continue / review what's done / replan
- 90+ days out: suggest `/vd-kill` or `/vd-spark` re-do (the idea may have matured)

**Doesn't pretend to know what happened during absence.** If the state is incomplete, the framework asks what the user remembers.

#### 7. Coleta Assistida de Features (Assisted Feature Collection)

New file `references/coleta-features-assistida.md`.

The `/vd-plan` command used to ask "list 10 testable features." This broke leigos (didn't know what "testable" meant, froze, listed 23 random things, gave up).

New behavior: `/vd-plan` opens with 3 conversational questions:

1. **Main flow** — "tell me step-by-step how you'd imagine someone using the product"
2. **What you don't want** — "what scares you, or what does the competitor get wrong"
3. **What would make you say 'this isn't for me'** — "what scenario would make users leave"

IA extracts features from narrative, marks MVP vs post-MVP vs anti-scope, writes human-readable "testable when [observable]" for each.

If user insists on dumping everything at once, IA captures all, **organizes by AI** into the 3 dimensions, and presents clean.

### Backward compatibility

- `brasil.md` removal: BR-related context now lives inside `modo-pais.md`'s BR section. **No code or behavior changes** for Brazilian projects.
- Onboarding only runs first session. Existing projects see no tour.
- Validation reinforcement in `/vd-check` doesn't break existing flow — it just refuses to accept "tá" as approval.
- Recap requires `ultima_sessao_em` field. `/vd-close` adds it automatically from v3.4.0 forward. Older projects without the field get recap by default.
- Feature collection only activates in `modo_usuario: leigo`. Technical mode unchanged.

1. **Reconhecimento honesto** — "não vou fingir que a gente já sabe como a ideia vira app. Isso aqui é construção em etapas, com você decidindo."
2. **Mapa visual das 5 fases** — diagrama ASCII mostrando o caminho.
3. **Escolha binária pra começar** — "tá começando do zero ou tem código bagunçado pra consertar?"

Depois da terceira mensagem, framework marca `onboarding_completo: true` no estado e **nunca** repete.

**Por que 3 e não 10:** leigos não leem parede de texto. 3 mensagens respeita orçamento de atenção.

#### 3. Painel Visual de Progresso

- Nome do projeto no topo
- Trilha (Verde / Vermelha)
- Barra de progresso (5 fases totais) com fase atual destacada
- Checklist de fases com estado (✅ feita / ➔ atual / ⏸ futura)
- Sub-tarefa atual + número / total
- Último marco alcançado (do post-mortem)

**Não substitui o estado** — o painel é leitor, não fonte de verdade.

#### 4. Decisão Confiante por Padrão

Modificação em `SKILL.md` → "Regras universais".

Leigos tendem a dizer "escolhe você" quando enfrentam opções técnicas Tipo 1. Framework respeitava como delegação mas **não registrava justificativa** — levando a decisões arrependidas depois.

Novo comportamento:

1. IA apresenta opções Tipo 1 em linguagem simples
2. Se usuário fala "escolhe você" / "tanto faz" / "confio":
   - IA escolhe a opção recomendada pelo Red Team
   - Justifica em 1 linha
   - Registra no Decision Log: "decisão Tipo X delegada pra IA por usuário em [data]"
   - Continua
3. Se usuário fala "quero entender melhor":
   - IA explica trade-off em termos leigos com 1 analogia se útil
   - Não pergunta de novo a menos que peça

**Registra delegação explicitamente** — pra sessões futuras saberem que o usuário não entendeu a decisão original.

#### 5. Validação Anti-"Tá"

Modificação no `/vd-check`.

Leigos falam "tá" / "deve tá" / "acho que sim" pra encerrar logo. Aceitar como aprovação = colocar feature não testada de verdade no ar.

Novo protocolo:

1. `/vd-check` pergunta coisa observável específica:
   > "Você clicou em '[botão X]', preencheu os campos, e viu [resultado visível específico] aparecer? Sim ou não — sem achismo."
2. Se usuário responder vago:
   - **NÃO aceita como aprovação**
   - Reformula: "Isso aconteceu AGORA, na sua tela, há alguns minutos?"
   - Se ainda assim vago → **modo DEBUG** (pede print, log, ou observação específica)
3. Só confirmação explícita move sub-tarefa pra "pronta"

**Protege o framework contra falsos positivos** que acumulariam em "o sistema funciona mas eu não consigo verificar de verdade".

#### 6. Recap Automático

Novo arquivo `references/recap-automatico.md`.

Leigos somem por dias ou semanas. Quando voltam, framework lia o estado e perguntava "onde quer continuar?" — frio, mecânico, inútil.

Novo comportamento no `/vd-status` quando `data_atual - ultima_sessao_em ≥ 7 dias`:

- Expande painel em **recap estendido** (bem-vindo de volta + último estado + decisões tomadas + persona + opções)
- Apresenta 3 opções explícitas: continuar / revisar o que tá pronto / replanejar
- 90+ dias fora: sugere `/vd-kill` ou refazer `/vd-spark` (ideia pode ter amadurecido)

**Não finge saber o que aconteceu durante ausência.** Se o estado tá incompleto, framework pergunta o que o usuário lembra.

#### 7. Coleta Assistida de Features

Novo arquivo `references/coleta-features-assistida.md`.

Comando `/vd-plan` antes pedia "liste 10 features testáveis". Quebrava leigos (não sabiam o que era "testável", travavam, listavam 23 coisas aleatórias, desistiam).

Novo comportamento: `/vd-plan` abre com 3 perguntas conversacionais:

1. **Fluxo principal** — "me conta passo-a-passo como você imagina alguém usando o produto"
2. **O que você não quer** — "o que te dá medo, ou o que o concorrente faz errado"
3. **O que faria você falar 'isso aqui não é pra mim'** — "que cenário faria o usuário sair"

IA extrai features da narrativa, marca MVP vs pós-MVP vs anti-escopo, escreve "testável quando [observável]" em linguagem humana.

Se usuário insiste em jogar tudo de uma vez, IA captura tudo, **organiza por IA** nas 3 dimensões e devolve limpo.

### Compatibilidade
