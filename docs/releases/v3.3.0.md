## 🇺🇸 English

**Release date:** 2026-07-10
**Scope:** VibeDev core (v1.4.0)

This release closes the loop for the layman user. v3.1.0 solved "I don't understand" (glossary, cost, kill). v3.2.0 solved "I'm exhausted / I lost control" (time, creep, recognition). v3.3.0 solves "I have an idea / I'm launched".

The 9 gaps originally identified for layman users are now **fully addressed** across three minor releases.

### What's in this release

#### 1. `/vd-spark` — Pre-Phase-1 Discovery

New command in `SKILL.md` + `vibedev/references/discovery-leigo.md`.

When a layman arrives with a vague idea (1-2 sentences, no clear persona, no clear success metric), the formal `/vd-start` is too dense. `/vd-spark` runs a 4-round conversation that extracts:

1. **The idea** — "tell me in 1 sentence what you're trying to build"
2. **For whom** — "who would use this? It can be yourself, a friend, a type of customer — anyone"
3. **What changes for that person** — "what can they do AFTER using this that they couldn't before?"
4. **How will you know it worked** — "what concrete signal makes you think 'it worked'?"

Output: `discovery_brief` block in `PROJECT_STATE.md` that **feeds** the formal `/vd-start`. The 3 diagnostic questions become pre-digested.

**Auto-activation:** if the project's first user message has < 3 sentences OR is missing "for whom" / "resolves" / "how to measure", `/vd-spark` runs before `/vd-start`. Silent for users who arrive with clear scope.

**Crucially:** doesn't replace `/vd-start`. Precedes it. No framework terminology surfaced — pure conversation, not an interview.

#### 2. `/vd-launch` — Communication Blocks for Launch Day

New command in `SKILL.md` + `assets/launch/launch-brief-template.md` + `assets/launch/checklist-pre-launch.md`.

Triggers only **after Phase 7 → 8 gate** (production is stable). The framework believes most launches fail because the product was launched during construction, so `/vd-launch` is **gated** until the foundation is solid.

The command delivers 6 ready-to-edit blocks:

- **30-second elevator pitch** (speech format, not text)
- **3 tweet variants** (problem / transformation / hook)
- **1 LinkedIn post** (~1500 chars, professional but human)
- **Landing page structure** (not copy — blocks with intent)
- **Personal email** for first 10 beta users (no newsletter tone)
- **Private post-mortem** (3 honest reflections the user reads later)

Plus a **24h pre-launch checklist** covering legal/LGPD, security, functionality, performance, content, operations, and communications.

**What it is NOT:** the IA does not build the landing page. Delivers blocks for the human to assemble and edit.

#### 3. Modo Brasil — Auto-activated Brazilian Context

New file `vibedev/references/brasil.md` + auto-activation rule in `SKILL.md`.

When session is pt-BR (default in Mavis) or project has BR context (BRL, Pix, .com.br, GMT-3), the framework loads local context:

- **BRL as primary currency** for cost simulations
- **Pix-first** payment suggestion (Stripe second when needed)
- **LGPD explicit** in security checks (via VibeShield handoff)
- **Brazilian hosting** options surfaced (Locaweb, King.host, AWS sa-east-1, etc.)
- **pt-BR UI strings** in all examples
- **DD/MM/AAAA dates**, GMT-3 timezone
- **NF-e / CNPJ / CPF** reminders for paid apps
- **Cultural context** (Pix as default expectation, support in pt-BR)

**Disclaimer present in `brasil.md`:** this is **not legal/accounting advice**. For serious projects, hire a Brazilian lawyer and accountant who work with startups/digital products. The file is the **starting point for asking the right questions**, not a compliance checklist.

### Backward compatibility

- `/vd-spark` is purely additive — skips when user intent is already clear.
- `/vd-launch` triggers only after Phase 7 → 8 gate. Does nothing during construction.
- Modo Brasil auto-activates but user can opt out ("modo Brasil off" or working on international project).
- New templates live under `assets/launch/` — don't conflict with existing state templates.

## 🇧🇷 Português

**Data do release:** 2026-07-10
**Escopo:** VibeDev core (v1.4.0)

Esse release fecha o ciclo pro usuário leigo. v3.1.0 resolveu "não entendo" (glossário, custo, kill). v3.2.0 resolveu "tô exausto / perdi o controle" (tempo, creep, reconhecimento). v3.3.0 resolve "tenho a ideia / tô no ar".

Os 9 buracos identificados pra leigos estão agora **100% endereçados** em 3 releases minor.

### O que tem nesse release

#### 1. `/vd-spark` — Discovery pré-Fase 1

Novo comando na `SKILL.md` + `vibedev/references/discovery-leigo.md`.

Quando o leigo chega com ideia vaga (1-2 frases, sem persona clara, sem métrica de sucesso), o `/vd-start` formal trava. `/vd-spark` roda 4 rodadas curtas de conversa que extraem:

1. **A ideia** — "me conta em 1 frase o que você tá querendo construir"
2. **Pra quem** — "quem ia usar isso? Pode ser você, amigo, tipo de cliente — qualquer um vale"
3. **O que muda pra essa pessoa** — "o que ela consegue fazer DEPOIS de usar, que antes não conseguia?"
4. **Como vai saber que deu certo** — "qual sinal concreto te faz pensar 'funcionou'?"

Saída: bloco `discovery_brief` no `PROJECT_STATE.md` que **alimenta** o `/vd-start` formal. As 3 perguntas densas de diagnóstico ficam pré-mastigadas.

**Ativação automática:** se a primeira mensagem do usuário tem < 3 frases OU falta "pra quem" / "resolve" / "como medir", `/vd-spark` roda antes do `/vd-start`. Silencioso pra quem chega com escopo claro.

**Crucial:** não substitui `/vd-start`. Precede ele. Sem mencionar framework, sem soar como entrevista.

#### 2. `/vd-launch` — Blocos de Comunicação pro Dia de Lançamento

Novo comando na `SKILL.md` + `assets/launch/launch-brief-template.md` + `assets/launch/checklist-pre-launch.md`.

Aciona **só após gate Fase 7 → 8** (produção estável). O framework acredita que a maioria dos lançamentos falha porque foi feito durante construção, então `/vd-launch` tem **gate rígido**.

Entrega 6 blocos prontos pra editar:

- **Elevator pitch de 30 segundos** (formato fala, não texto)
- **3 variantes de tweet** (problema / transformação / gancho)
- **1 post LinkedIn** (~1500 caracteres, profissional mas humano)
- **Estrutura de landing page** (não cópia pronta — blocos com intenção)
- **E-mail pessoal** pros primeiros 10 beta users (sem tom de newsletter)
- **Post-mortem privado** (3 reflexões honestas que você lê depois)

+ **Checklist pré-lançamento 24h** cobrindo legal/LGPD, segurança, funcionalidade, performance, conteúdo, operação e comunicação.

**O que NÃO é:** a IA não constrói a landing page. Entrega blocos pro humano editar.

#### 3. Modo Brasil — Contexto BR Auto-Ativado

Novo arquivo `vibedev/references/brasil.md` + regra de auto-ativação na `SKILL.md`.

Quando sessão é pt-BR (default no Mavis) ou projeto tem contexto BR (BRL, Pix, .com.br, GMT-3), o framework carrega contexto local:

- **BRL como moeda primária** pra simulações de custo
- **Pix-first** na sugestão de pagamento (Stripe em segundo quando precisar)
- **LGPD explícita** nas checagens de segurança (via handoff VibeShield)
- **Hospedagem BR** surfaced (Locaweb, King.host, AWS sa-east-1, etc)
- **UI strings pt-BR** em todos os exemplos
- **Datas DD/MM/AAAA**, fuso GMT-3
- **Lembretes de NF-e / CNPJ / CPF** pra apps pagos
- **Contexto cultural** (Pix como expectativa default, suporte em pt-BR)

**Disclaimer no próprio arquivo:** **não é assessoria jurídica nem contábil**. Pra projeto sério, contrate advogado e contador BR que atuam com startups/produto digital. O arquivo é o **começo das perguntas certas**, não checklist de conformidade.

### Compatibilidade

- `/vd-spark` puramente aditivo — pula quando intenção já tá clara.
- `/vd-launch` só dispara após gate Fase 7 → 8. Não faz nada durante construção.
- Modo Brasil auto-ativa mas tem opt-out ("modo Brasil off" ou trabalho em projeto internacional).
- Novos templates em `assets/launch/` — não conflitam com templates de estado existentes.
