## 🇺🇸 English

**Release date:** 2026-07-08
**Scope:** VibeDev core (v1.2.0)

This release strengthens VibeDev for the most common user of vibe-coded projects in 2026: the person with a brilliant idea who watched an Instagram reel saying "you can build an app with AI" and downloaded Cursor or Claude Code the same night.

We started from a candid reflection on what was still missing in the framework for non-engineers. The list narrowed to 9 gaps. This release ships the first 3 (the high-impact tier). The remaining 6 are queued for v3.2.0 and v3.3.0.

### Acknowledgement / Origin of the work

The **Glossário Ativo** design draws on the same epistemic discipline surfaced by **[Sandeco Macedo](https://github.com/sandeco)** (Instituto Federal de Goiás) in his preprint *"Stop Hand-Holding Your Coding Agent: Engineering the Loops that Replace Step-by-Step Prompting"* ([arXiv:2607.00038](https://arxiv.org/abs/2607.00038), June 2026). Macedo's framework distinguishes between **prompt** (single instruction), **context engineering** (information around the model), **harness** (the agent's environment), and **loop** (the autonomous corrective cycle).

While VibeDev doesn't import Loop Engineering wholesale — VibeDev governs the full project lifecycle, and loops are a specific operation within Phase 6 (Construction) or Phase 8 (Operation) — the cross-pollination is real. Especially the **5-level verification ladder** (deterministic → rule → field truth → model-as-judge → human checkpoint), which mirrors the **/vd-check** validation protocol almost beat for beat. And the **anti-patterns section** of Macedo's work informed the Cost Guard design (avoiding "while-true around a stranger" loops where the model just agrees with itself).

If you build agent systems or design loops, Macedo's paper is worth the 30 minutes. [arXiv:2607.00038](https://arxiv.org/abs/2607.00038).

### What's in this release

#### 1. Layman Glossary (auto-active)

New file: `vibedev/references/glossario-leigo.md`

When a project has `modo_usuario: leigo` in its `PROJECT_STATE.md`, the AI now silently translates technical jargon into plain language in EVERY output. The glossary covers:

- Construction & code (~30 terms)
- Database (~20 terms)
- Auth & security (~20 terms)
- Server & infra (~20 terms)
- Payment & business (~15 terms)
- Loop Engineering vocabulary (~10 terms)
- VibeDev process vocabulary (~15 terms)

**Design choices:**

- Names of technologies (Python, Postgres, Stripe, etc.) stay untranslated — the problem is not the name, it's the concept behind it.
- Unlisted terms encountered mid-conversation are translated inline and appended to the glossary silently.
- The IA passes a mental "adult without engineering background could read this and understand" test before emitting any sentence in layman mode.

#### 2. Cost Guard (universal rule)

New file: `vibedev/references/estimativa-custos.md`

A mandatory gate before Phase 3 → Phase 4 promotion. The state file must contain a `custo_mensal_estimado` table with 3 user tiers (1-10, 10-100, 100-1000), in BRL/month.

Reference table covers 6 categories:

- Landing page
- Institutional site (3-10 pages)
- Internal app / private dashboard
- Public B2C SaaS (freemium, login, payment)
- Messenger bot (Telegram, WhatsApp)
- Mobile app (iOS + Android)

**Behavior:**

- AI must say "let's see what this will cost per month to keep running" before any stack choice.
- If any tier exceeds user budget → back to Phase 2, reduce scope, recalculate.
- If user wants to skip → register the decision in the Decision Log. Human > rule.

**Honest disclaimer:** values are order-of-magnitude estimates. Real quotation requires entering each service's site. Not included: human maintenance time, taxes, support cost, design/copy.

#### 3. `/vd-kill` — Authorized termination

New file: `vibedev/assets/IDEA_LOG-template.md`

Recognition that **not every idea is worth continuing**. Exit with dignity is as important as finishing. The anti-guilt command.

When to use:

- User explicitly says they want to quit ("don't want anymore", "I'm killing it", "not worth it").
- Cost estimate exceeds budget in every tier and the user doesn't want to reduce scope.
- Kill criteria from Phase 1 triggered.
- Project has been idle 90+ days with no new decisions.

What it does:

1. Confirms decision with one clear sentence (waits for explicit yes).
2. Creates/updates `IDEA_LOG.md` at project root with 3 reflective questions:
   - What problem did I really think this idea solved?
   - What did I learn trying (3 bullets)?
   - Is there a smaller path I could try first, or is this a full stop?
3. Renames `PROJECT_STATE.md` → `IDEA_LOG_<name>_<date>.md` (preserves history).
4. Confirms: "Archived. When you want to start from zero again, run `/vd-start`. The learning is saved."

**What it ISN'T:** failure. It's recognition that something isn't worth it **right now**. Nothing gets deleted — all artifacts land in `IDEA_LOG.md`.

### Backward compatibility

- Projects in `modo_usuario: tecnico` (or unset): unaffected. All additions are leigo-mode-only.
- New `custo_mensal_estimado` field: mandatory for new Layman Mode projects. Existing ones adopt it on next phase transition.
- `/vd-kill` is a new command. Doesn't modify any existing command.
- Glossary only triggers under `modo_usuario: leigo`. Technical mode outputs identical to v3.0.0.

### O que tem nesse release

#### 1. Glossário Ativo (auto-ativo)

Arquivo novo: `vibedev/references/glossario-leigo.md`

Quando o projeto tem `modo_usuario: leigo` no `PROJECT_STATE.md`, a IA agora traduz silenciosamente jargão técnico pra linguagem humana em TODA saída. O glossário cobre:

- Construção & código (~30 termos)
- Banco de dados (~20 termos)
- Autenticação & segurança (~20 termos)
- Servidor & infra (~20 termos)
- Pagamento & negócio (~15 termos)
- Vocabulário Loop Engineering (~10 termos)
- Vocabulário de processo VibeDev (~15 termos)

**Decisões de design:**

- Nomes de tecnologia (Python, Postgres, Stripe, etc.) ficam sem traduzir — o problema não é o nome, é o conceito por trás.
- Termos não listados que aparecem no meio da conversa são traduzidos inline e adicionados ao glossário em silêncio.
- A IA passa mentalmente pelo teste "um adulto sem formação em TI lê e entende sozinho" antes de emitir qualquer frase em modo leigo.

#### 2. Guarda de Custo (regra universal)

Arquivo novo: `vibedev/references/estimativa-custos.md`

Gate obrigatório antes de promover Fase 3 → 4. O arquivo de estado precisa ter a tabela `custo_mensal_estimado` com 3 portes (1-10, 10-100, 100-1000), em BRL/mês.

Tabela de referência cobre 6 categorias:

- Landing page
- Site institucional (3-10 páginas)
- App interno / dashboard privado
- SaaS B2C público (freemium, login, pagamento)
- Bot em mensageria (Telegram, WhatsApp)
- App mobile (iOS + Android)

**Comportamento:**

- IA precisa dizer "vamos ver o que isso vai custar por mês pra rodar" antes de qualquer escolha de stack.
- Se algum porte estourar o orçamento do usuário → volta pra Fase 2, reduz escopo, recalcula.
- Se o usuário quiser pular → registra no Decision Log. Humano > regra.

**Aviso honesto:** valores são estimativas de ordem de grandeza. Cotação real exige entrar no site de cada serviço. Não inclui: tempo humano de manutenção, impostos, custo de suporte, design/copy.

#### 3. `/vd-kill` — Encerramento autorizado

Arquivo novo: `vibedev/assets/IDEA_LOG-template.md`

Reconhecimento de que **nem toda ideia vale continuar**. Sair com dignidade é tão importante quanto terminar. O comando anti-culpa.

Quando usar:

- Usuário declara explicitamente que quer desistir ("não quero mais", "vou matar", "não vale a pena").
- Estimativa de custo estourou o orçamento em todo porte e o usuário não quer cortar escopo.
- Kill criteria da Fase 1 foi atingido.
- Projeto tá parado há 90+ dias sem decisão nova.

O que faz:

1. Confirma com 1 frase clara (espera yes explícito).
2. Cria/atualiza `IDEA_LOG.md` na raiz do projeto com 3 perguntas reflexivas:
   - Qual problema eu achava de verdade que essa ideia resolvia?
   - O que eu aprendi tentando (3 bullets)?
   - Tem um caminho menor que eu poderia tentar antes, ou encerra de vez?
3. Renomeia `PROJECT_STATE.md` → `IDEA_LOG_<nome>_<data>.md` (preserva histórico).
4. Confirma: "Arquivado. Quando quiser voltar do zero, faça `/vd-start`. O aprendizado ficou salvo."

**O que NÃO é:** fracasso. É reconhecimento de que algo não vale **agora**. Nada é apagado — tudo vai pro `IDEA_LOG.md`.

### Compatibilidade

- Projetos em `modo_usuario: tecnico` (ou vazio): sem mudança. Tudo que é novo é só pra modo leigo.
- Campo novo `custo_mensal_estimado`: obrigatório só pra projetos novos em Modo Leigo. Os existentes adotam na próxima transição de fase.
- `/vd-kill` é um comando novo. Não modifica nenhum comando existente.
- Glossário só ativa sob `modo_usuario: leigo`. Saída do modo técnico idêntica ao v3.0.0.
